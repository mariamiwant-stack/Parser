# Email Crawler (Python + Playwright)

Современный и отказоустойчивый парсер email-адресов предприятий по типу бизнеса и городам России.

Теперь поддерживаются **два режима**:
- **Windows GUI (Tkinter)** — основной режим для удобной работы.
- **CLI** — для автоматизации/скриптов.

## Возможности

- Поиск сайтов предприятий в поисковой выдаче по пользовательским фразам.
- Поддержка одного города, списка городов или всех топ‑16 городов России.
- Извлечение email с главной и контактных страниц.
- Реальное время сохранения: JSON обновляется сразу после каждого найденного email/изменения статистики.
- Надёжные ретраи с экспоненциальной задержкой на сетевых ошибках и HTTP 4xx/5xx.
- Обнаружение капчи (без попытки автоматического решения).
- Подробное логирование (`logs/crawler.log`).
- Защита от дублей email в рамках одного города.
- Неперезапись существующих файлов (автодобавление timestamp к имени).

## Установка

```bash
python -m venv .venv
# Windows PowerShell:
.\.venv\Scripts\Activate.ps1
pip install -r requirements.txt
python -m playwright install chromium
```

## Запуск GUI (Windows)

```bash
python gui.py
```

Что можно настроить в интерфейсе:
- тип бизнеса;
- поисковые фразы через запятую;
- режим городов: один / группа / все 16;
- задержки, timeout, retries;
- headless режим;
- путь к JSON-результатам и логам.

После нажатия **«Запуск»** в окне показывается живой прогресс (поисковый запросы, сайты, найденные email, ошибки).

## CLI (дополнительно)

### 1) Один город

```bash
python crawler.py \
  --business-type "автосервисы" \
  --queries "автосервисы,автосервисы лучшие" \
  --city "Москва" \
  --pages 3 \
  --max-sites 40 \
  --search-delay 2 \
  --site-delay 1.5
```

### 2) Группа городов

```bash
python crawler.py \
  --business-type "цветочные магазины" \
  --queries "цветочные магазины,доставка цветов" \
  --cities "Москва,Санкт-Петербург,Казань" \
  --group-name "group_1" \
  --pages 2 \
  --max-sites 25
```

### 3) Все 16 городов

```bash
python crawler.py \
  --business-type "стоматологии" \
  --queries "стоматологии,стоматология круглосуточно" \
  --all-cities \
  --pages 2 \
  --max-sites 20
```

## Поведение при ошибках

- Любая ошибка запроса/парсинга логируется и не останавливает процесс.
- Капча распознаётся, событие логируется как `captcha_detected`, запрос пропускается.
- Для недоступных сайтов выполняется retry (экспоненциальная задержка).
- Ошибки записи в JSON минимизированы атомарной записью через `.tmp` файл и замену.
