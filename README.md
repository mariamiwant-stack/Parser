# Email Crawler (Python + Playwright)

Современный и отказоустойчивый CLI‑краулер для поиска email-адресов предприятий по типу бизнеса и городам России.

## Возможности

- Поиск сайтов предприятий в поисковой выдаче по пользовательским фразам.
- Поддержка одного города, списка городов или всех топ‑16 городов России.
- Извлечение email с главной и контактных страниц.
- Реальное время сохранения: JSON обновляется **сразу после каждого найденного email/изменения статистики**.
- Надёжные ретраи с экспоненциальной задержкой на сетевых ошибках и HTTP 4xx/5xx.
- Обнаружение капчи (без попытки автоматического решения).
- Подробное логирование (`logs/crawler.log`).
- Защита от дублей email в рамках одного города.
- Неперезапись существующих файлов (автодобавление timestamp к имени).

## Установка

```bash
python -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
python -m playwright install chromium
```

## Быстрый старт

### 1) Один город

```bash
python crawler.py \
  --business-type "автосервисы" \
  --queries "автосервисы,автосервисы лучшие" \
  --city "Москва" \
  --pages 3 \
  --max-sites 40 \
  --search-delay 2 \
  --site-delay 1.5
```

Ожидаемый файл:

- `output/emails_moscow_автосервисы.json`

### 2) Группа городов

```bash
python crawler.py \
  --business-type "цветочные магазины" \
  --queries "цветочные магазины,доставка цветов" \
  --cities "Москва,Санкт-Петербург,Казань" \
  --group-name "group_1" \
  --pages 2 \
  --max-sites 25
```

Ожидаемый файл (имя для группы):

- `output/emails_group_1_цветочные_магазины.json`

### 3) Все 16 городов

```bash
python crawler.py \
  --business-type "стоматологии" \
  --queries "стоматологии,стоматология круглосуточно" \
  --all-cities \
  --pages 2 \
  --max-sites 20
```

## Ключевые параметры CLI

- `--business-type` — тип бизнеса (обязательный).
- `--queries` — поисковые фразы через запятую (обязательный).
- `--city` / `--cities` / `--all-cities` — выбор города(ов), взаимоисключающие флаги.
- `--pages` — страницы поисковой выдачи (1..10).
- `--max-sites` — максимум сайтов на город.
- `--search-delay` — задержка между запросами к поиску.
- `--site-delay` — задержка между запросами к сайтам.
- `--timeout` — таймаут загрузки страницы.
- `--retries` — число повторов при ошибках.
- `--retry-base-delay` — база для экспоненциальной задержки.
- `--headful` — отключает headless режим.
- `--log-file` — путь к лог-файлу.
- `--output-dir` — путь к JSON результатам.

## Поведение при ошибках

- Любая ошибка запроса/парсинга логируется и не останавливает процесс.
- Капча распознаётся, событие логируется как `captcha_detected`, запрос пропускается.
- Для недоступных сайтов выполняется retry (экспоненциальная задержка).
- Ошибки записи в JSON минимизированы атомарной записью через `.tmp` файл и замену.
